# AGENTS.md

CatWing brand identity: source-of-truth for colors, typography, logo variants, iconography, voice, components, interaction patterns. Consumed by humans (designers, marketing) and agents (Claude Code, Codex) building UI in the CatWing repos.

## What lives here

| Path | Purpose |
|------|---------|
| `tools/build.py` | Compiles `index.html`, `brand.html`, `design.html`, `resources.html` (self-contained, base64-embedded fonts + logos) |
| `src/style.css`, `src/script.js` | Wiki page styling + interaction (search, theme toggle) |
| `logo/` | All raster logo variants + bundled streamlit assets. Every asset referenced by `tools/build.py::LOGO_DIR_KEYS` lives here |
| `logo/fonts/` | Cached font files (gitignored — regenerated by `tools/build.py`) |
| `.agents/skills/brand-check/` | Skill (canonical): lint a UI surface for off-brand colors, typography, logo misuse |
| `.agents/skills/brand-asset/` | Skill (canonical): inline color tokens + logo path reference for agent context |
| `.claude/skills/` | Auto-generated mirror of `.agents/skills/` — never edit directly |
| `tools/hooks/sync_skills.py` | Mirrors `.agents/skills/` → `.claude/skills/`; rejects direct edits to the mirror |
| `tools/hooks/check_skills_parity.py` | Always-on guard: fails commit if the two trees drift |
| `.github/workflows/pages.yml` | Builds + deploys the four HTML pages to GitHub Pages on push to `main` |

## Rules

- This repo is the canonical brand source. Do **not** duplicate brand instructions in `catwing/`, `catwing-app/`, `catwing-rest-api/`, etc. — link here instead.
- `tools/build.py` must produce a self-contained build with no path arguments. Do not re-introduce `--app-dir` or upward-walking `resolve_app_dir()` lookups; bundle copies into `logo/` instead.
- Keep `LOGO_DIR_KEYS` and the section builders (`_sec_logos`, `_sec_colors`, `_sec_typography`, …) consistent — adding a key without a section consumer is dead asset weight.
- Never commit compiled `*.html` outputs or `.fonts/` cache. CI rebuilds on every push.
- Two assets in `logo/` (`streamlit_logo.png`, `streamlit_favicon.ico`) are bundled copies of `catwing-app/streamlit_apps/assortment_review/static/img/{logo.png,favicon.ico}`. If those upstream files change, refresh the bundles in the same change and bump `VERSION` in `tools/build.py`.
- Any color or font added to `src/style.css` must also appear in the brand guide (`tools/build.py::COLORS` or the typography section) — otherwise it is unauthorized drift.
- `.agents/skills/` is canonical; `.claude/skills/` is a generated mirror. Edit `.agents/skills/` only — `tools/hooks/sync_skills.py` regenerates the mirror, and `tools/hooks/check_skills_parity.py` blocks any commit where the two trees diverge.
- Skill prompts must reference this file (or a specific section of a generated HTML) for canonical values; do not hard-code colors inside skill markdown that could go stale.

## Commands

```bash
python tools/build.py                  # build + open index.html in browser
python tools/build.py --no-open        # build only (used by CI)
python tools/build.py --fresh-fonts    # force re-download of Google Fonts
```

## Verification

- `tools/build.py` exits 0 and writes 4 HTMLs (each > 100 KB once fonts + logos are embedded).
- `index.html` renders the wiki landing page with working navigation to the other three.
- `brand.html` shows the cover, philosophy, logo grid, full color palette, typography specimens, iconography, voice, components, interactions, states.
- Open the four HTMLs directly via `file://` — they must render fully offline (no external font/image fetches).

## Known drift

- The wiki's `_load_fonts()` step downloads from Google Fonts on first build and caches under `.fonts/`. Offline-only environments should commit a populated `.fonts/` once or vendor the woff2 files explicitly. (Currently gitignored.)
- `src/style.css` and `tools/build.py` both encode color values; the brand guide is the source of truth, but a stylesheet change without a `tools/build.py::COLORS` update will silently diverge. A future check should diff the two.
